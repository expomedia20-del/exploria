تاریخ: ۱۴۰۵/۰۴/۱۳ - 2026-07-04

این نسخه، ارزیابی قبلی مورخ 2026-07-02 را با توجه به پیشرفت‌های جدید پروژه بازبینی می‌کند: تکمیل نسبی چرخه اسپانسر، تعریف نقش‌ها و دسترسی‌ها، پنل مدیر مکان، پنل مدیر رواق/زون، اتصال عملی پاداش/گنج اسپانسری و آماده‌سازی مسیر دمو.

آخرین commit مبنا: 107658a - docs: add current handoff snapshot

خلاصه مدیریتی

اکسپلوریا از یک ایده تجربه‌محور به یک محصول عملیاتی‌تر نزدیک شده است. اکنون پروژه فقط «بازی گنج» نیست؛ بلکه لایه عملیاتی برای اتصال مکان، فروشگاه، اسپانسر، نمایشگر، ماموریت، پاداش و داده رفتاری کاربر دارد. این تغییر ارزش بیزینسی پروژه را بالا برده و آن را برای گفت‌وگو با مکان‌ها، حامیان مالی و سرمایه‌گذار قابل دفاع‌تر می‌کند.

پیشرفت‌های کلیدی پس از نسخه قبل

  • تعریف عملیاتی نقش‌ها و scopeها: مدیر پروژه مکانی، مدیر مکان، مدیر رواق/زون، مدیر هاب، مدیر فروشگاه، اسپانسر و مدیر نمایشگر.
  • افزودن قالب‌های آماده تخصیص دسترسی برای اکوپارک: مدیر مکان، رواق، هاب ورودی، کافه اکو، مدیر نمایشگر و مدیر پروژه.
  • ایجاد پنل read-only مدیر مکان در مسیر /venue/dashboard برای مشاهده کمپین، هاب‌ها، شرکا، نمایشگرها، تبلیغات، پاداش‌ها و گنج‌ها.
  • ایجاد مسیر مستقل پنل مدیر رواق/زون در /ravaq/dashboard با استفاده از hub scope پشت صحنه.
  • تقویت چرخه اسپانسر: پیشنهاد، اصلاح، تایید بسته اجرایی، اتصال به ماموریت/پاداش/گنج و ثبت سهم واحدها.
  • ثبت سند handoff برای ادامه کار بدون گم شدن وضعیت پروژه.
برداشت مهم: پروژه اکنون برای مذاکره فقط نیازمند توضیح ایده نیست؛ می‌توان «جریان عملیاتی واقعی» را نشان داد.

پیش‌بینی موفقیت و اولویت بعدی
