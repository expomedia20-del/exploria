# 26 — EcoPark Demo Readiness Check

## هدف

این سند مرحله اول پایدارسازی دموی کامل اکوپارک را ثبت می‌کند. هدف این مرحله این است که قبل از اجرای دموی فروش‌پذیر، زنجیره عملیاتی زیر با یک دستور قابل کنترل باشد:

`مکان فعال → کمپین فعال → QR → ماموریت‌ها → پاداش‌ها → گنج → واحدهای تجاری → اسپانسر → ردیابی سهم/موجودی → پنل‌های مدیریتی`

## دستور اجرا

```bash
php artisan exploria:demo-readiness
```

برای خروجی JSON:

```bash
php artisan exploria:demo-readiness --json
```

برای بررسی مکان دیگر:

```bash
php artisan exploria:demo-readiness --venue=ecopark-abbasabad
```

## معیارهای کنترل

چک آمادگی دمو این بخش‌ها را بررسی می‌کند:

- فعال بودن مکان پایلوت و پروفایل آن
- وجود کمپین فعال برای مکان
- وجود QR فعال متصل به کمپین
- وجود زنجیره ماموریت چندمرحله‌ای
- اتصال حداقل یک گنج به کمپین یا ماموریت
- وجود حساب‌های شریک، فروشگاه و اسپانسر
- اتصال واحدها به هاب/رواق و عضویت در کمپین
- وجود پاداش‌های چندلایه، پاداش شریک و مشوق اسپانسری
- وجود ردیابی سهم و موجودی برای مشوق‌های قابل دریافت
- وجود نمایشگر عملیاتی
- وجود رواق تجاری فعال
- وجود دسترسی‌های مدیر رواق، فروشگاه و اسپانسر
- وجود مسیر پنل‌های مکان، رواق، هاب، اسپانسر و داشبورد اصلی

## تفسیر وضعیت

- `pass`: بخش آماده است.
- `warning`: دمو می‌تواند ادامه پیدا کند، اما برای اجرای کامل‌تر باید تکمیل شود.
- `fail`: زنجیره دمو ناقص است و باید قبل از نمایش اصلی رفع شود.

در داده دمو، دسترسی فعال `venue_executive` برای مدیر اجرایی اکوپارک و سهم موجودی پاداش‌های قابل دریافت ثبت می‌شود. بنابراین Seed استاندارد باید گزارش آمادگی را بدون Warning یا Fail تولید کند.

## تست

تست مرتبط:

```bash
php artisan test tests/Feature/Demo/EcoParkDemoReadinessTest.php
```

این تست هم Service را بررسی می‌کند و هم command خروجی JSON را اجرا می‌کند.
