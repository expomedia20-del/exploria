# مدل نقش‌ها، وظایف و عملیات روزانه اکسپلوریا

این سند مرجع مرحله بعدی کدنویسی نقش‌ها و دسترسی‌هاست. اصل طراحی این است که «نقش» و «دامنه دسترسی» از هم جدا باشند:

```text
Role = کاربر چه نوع مسئولیتی دارد
Scope = این مسئولیت روی کدام محدوده اعمال می‌شود
```

مثال‌ها:

- ادمین اصلی: `role = super_admin` و `scope = global`
- ادمین منطقه‌ای تهران: `role = regional_admin` و `scope = region:tehran`
- ادمین پروژه اکوپارک: `role = project_admin` و `scope = venue:ecopark`
- مدیر رواق: `role = hub_manager` و `scope = hub:ravaq`
- مدیر کافه: `role = shop_manager` و `scope = partner:cafe-eco`

## دامنه‌های دسترسی

| دامنه | کاربرد |
| --- | --- |
| `global` | کل اکوسیستم اکسپلوریا |
| `region` | استان، منطقه یا عاملیت منطقه‌ای |
| `venue` | یک مکان پروژه مثل اکوپارک، برج میلاد یا ارم |
| `project` | یک پروژه اجرایی یا قرارداد مشخص در یک یا چند مکان |
| `hub` | هاب یا رواق داخل مکان پروژه |
| `partner` | فروشگاه، واحد تجاری، غذایی، فرهنگی یا خدماتی |
| `campaign` | کمپین، مسیر، ماموریت یا عملیات میدانی مشخص |
| `display_network` | شبکه نمایشگرهای ثابت و سیار |
| `team` | مشارکت فردی، خانوادگی یا تیمی کاربر نهایی |

## نقش‌های داخلی تیم اکسپلوریا

| نقش | دامنه معمول | وظایف اصلی | برنامه عملیاتی روزانه |
| --- | --- | --- | --- |
| ادمین اصلی کل اکسپلوریا | `global` | مالکیت سیاست‌ها، دسترسی کامل، تاییدهای حساس، نظارت بر کل استان‌ها، مکان‌ها، هاب‌ها، فروشگاه‌ها، اسپانسرها، کمپین‌ها و نمایشگرها | بررسی سلامت کل سیستم، تاییدهای معطل، خطاهای بحرانی، گزارش اسپانسر/تبلیغات/پاداش، تعیین اولویت فردا |
| ادمین منطقه‌ای / عاملیت استانی | `region` | مدیریت استان یا منطقه، تایید پروژه‌ها و شرکای منطقه، کنترل عملکرد پیمانکاران و عملیات | بررسی مکان‌های فعال، کمپین‌ها، نیروی انسانی، درخواست‌های منطقه‌ای، ارسال گزارش روزانه به ادمین اصلی |
| ادمین پروژه مکانی / پیمانکار پروژه | `venue` یا `project` | اجرای یک یا چند پروژه مثل اکوپارک، برج میلاد، ارم، هماهنگی مدیر مکان، هاب‌ها، فروشگاه‌ها و تیم میدانی | بررسی آمادگی مکان، QRها، هاب‌ها، نمایشگرها، درخواست‌های فروشگاه/اسپانسر، جمع‌بندی ریسک‌های روز |
| مجری میدانی کمپین | `campaign` | اجرای کمپین در محل، کنترل QR، مسیر، صف، پاداش، گزارش مشکل و کمک به اجرای درست سناریو | بازدید نقاط تماس، ثبت مشکلات با عکس و موقعیت، کمک در ساعات شلوغ، گزارش پایان شیفت |
| یاریگر همراه کاشفان گنج | `campaign` | راهنمایی بازدیدکننده، خانواده یا تیم، کمک در اسکن، شروع ماموریت و رفع مشکلات ساده | آماده‌سازی ایستگاه راهنما، کمک به شروع ماموریت، ثبت سوالات و خطاهای تکراری، تحویل مشکلات حل‌نشده |
| مدیر تنظیم تبلیغات نمایشگرها | `display_network` | زمان‌بندی، کنترل سلامت و گزارش پخش روی نمایشگرهای ثابت و سیار، رفع تداخل تبلیغات | بررسی سلامت دستگاه‌ها، زمان‌بندی روز بعد، کنترل تداخل‌ها، خروجی گرفتن از خطاهای پخش و شواهد قابل تسویه |

## نقش‌های بیرونی و شرکای پروژه

| نقش | دامنه معمول | وظایف اصلی | برنامه عملیاتی روزانه |
| --- | --- | --- | --- |
| مدیر اجرایی مکان پروژه | `venue` | نماینده اجرایی مکان مثل اکوپارک یا شهربازی، کنترل ملاحظات عملیاتی، امنیتی و تجاری همان مکان | بررسی برنامه روز، محدودیت‌های مکان، گزارش هاب‌ها، تایید تغییرات اجرایی محلی |
| مدیر هاب / رواق | `hub` | مدیریت یک هاب مثل رواق، علمی، آموزشی، فرهنگی، غذایی یا تجاری، نظارت بر واحدهای زیرمجموعه | بررسی پنل هاب، وضعیت فروشگاه‌ها، موجودی پاداش، درخواست تبلیغ، تایید/رد/ارجاع موارد هاب |
| مدیر فروشگاه یا واحد تجاری | `partner` | مدیریت پروفایل، پیشنهادها، کدهای پاداش، موجودی، تبلیغات و خدمت‌رسانی به بازدیدکننده | بررسی کدهای پاداش، موجودی، وضعیت پیشنهادها، ثبت پیشنهاد یا تبلیغ جدید، گزارش مشکل به مدیر هاب |
| اسپانسر داخلی | `hub` یا `venue` | اسپانسری که تحت مدیریت هاب یا مکان فعالیت می‌کند و درخواست‌هایش از همان زنجیره عبور می‌کند | بررسی وضعیت تبلیغ و جایزه، پاسخ به یادداشت مدیر هاب یا ادمین، آماده‌سازی دارایی‌های اسپانسری |
| اسپانسر خارجی | `region` یا `global` | اسپانسر مستقل از مکان یا هاب که باید تحت اشراف ادمین اصلی و ادمین منطقه‌ای باشد | بررسی کمپین‌های فعال، ارسال درخواست جدید، پاسخ به یادداشت ادمین، بررسی KPI و شواهد تسویه |
| کاربر / بازدیدکننده / مشارکت‌کننده | `team` | شرکت در کمپین به صورت فردی، خانوادگی یا تیمی، انجام ماموریت، کشف گنج و دریافت پاداش | شروع یا ادامه مسیر، انجام ماموریت مجاز، بررسی امتیاز و پاداش، ثبت بازخورد یا مشکل |

## زنجیره نظارت پیشنهادی

```text
ادمین اصلی
  ادمین منطقه‌ای
    ادمین پروژه مکانی / پیمانکار
      مجری میدانی
        یاریگر کاشفان گنج
      مدیر تبلیغات نمایشگرها
      مدیر اجرایی مکان
        مدیر هاب / رواق
          مدیر فروشگاه یا واحد
          اسپانسر داخلی
    اسپانسر خارجی
کاربر نهایی مستقل از زنجیره مدیریتی است، اما فعالیت او در دامنه کمپین و تیم ثبت می‌شود.
```

## قواعد دسترسی کلیدی

1. ادمین اصلی همیشه دسترسی کامل دارد.
2. ادمین منطقه‌ای فقط محدوده استان یا منطقه خود را می‌بیند، مگر برای گزارش‌های تجمیعی مجاز.
3. ادمین پروژه فقط مکان‌ها یا پروژه‌هایی را می‌بیند که به او تخصیص داده شده‌اند.
4. مدیر اجرایی مکان کل همان مکان را می‌بیند، اما به مکان‌های دیگر دسترسی ندارد.
5. مدیر هاب فقط هاب‌های تخصیص‌داده‌شده و زیرمجموعه‌های همان هاب را می‌بیند.
6. مدیر فروشگاه فقط پروفایل، پیشنهادها، پاداش‌ها، تبلیغات و گزارش‌های واحد خودش را می‌بیند.
7. اسپانسر داخلی از مسیر هاب یا مکان تایید می‌شود.
8. اسپانسر خارجی از مسیر ادمین منطقه‌ای و ادمین اصلی تایید می‌شود.
9. مجری میدانی و یاریگر فقط ابزارهای اجرایی، ثبت وضعیت و گزارش مشکل دارند؛ دسترسی مالی یا سیاست‌گذاری ندارند.
10. مدیر نمایشگرها به زمان‌بندی، وضعیت پخش و شواهد پخش دسترسی دارد، اما تغییر سیاست تجاری اسپانسر یا پاداش با او نیست.

## تبدیل به کدنویسی

مرحله پیشنهادی بعدی:

1. حفظ نقش‌های فعلی برای جلوگیری از شکست ورود و 403.
2. افزودن جدول `user_access_scopes` برای اتصال کاربر به `scope_type` و `scope_id`.
3. نگاشت نقش‌های قدیمی به نقش‌های جدید:
   - `admin` -> `super_admin`
   - `operator` -> `field_operator` یا `display_ads_manager` بر اساس پنل
   - `hub_manager` -> `hub_manager`
   - `shop_partner` -> `shop_manager`
   - `sponsor` -> `internal_sponsor` یا `external_sponsor`
   - `visitor` -> `participant`
4. افزودن صفحه مدیریت دسترسی برای ادمین اصلی و ادمین منطقه‌ای.
5. اتصال منوی کناری، Route middleware و سرویس‌های داشبورد به مدل Role + Scope.

فایل کانفیگ عملیاتی این نقش‌ها در مسیر زیر ثبت شده است:

```text
config/exploria_roles.php
```
