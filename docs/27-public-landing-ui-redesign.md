# 27 — Public Landing UI Redesign

## هدف

این تغییر برای اصلاح نامیزانی صفحه عمومی اکسپلوریا انجام شد. نسخه قبلی Hero یک تصویر بزرگ شناور داشت که با تیتر اصلی تداخل بصری ایجاد می‌کرد و پیام صفحه را در نمای اول مبهم می‌کرد.

## اصلاحات انجام‌شده

- Hero از حالت تصویر شناور و absolute به layout دو ستونه کنترل‌شده تبدیل شد.
- تیتر اصلی کوتاه‌تر و مستقیم‌تر شد: تبدیل مکان به تجربه، فروش و گزارش.
- CTA اصلی به «شروع دموی اکوپارک» تغییر کرد تا مسیر دمو برای مخاطب روشن‌تر باشد.
- چهار هدف اصلی در نمای اول اضافه شد:
  - بازدیدکننده مسیر و پاداش می‌گیرد.
  - واحد عضو پیشنهاد و مصرف را می‌بیند.
  - اسپانسر جایزه و ROI دریافت می‌کند.
  - مدیر مکان نمای کل اجرا را دارد.
- Mockup دمو داخل یک قاب محصول منظم قرار گرفت و دیگر با متن اصلی هم‌پوشانی ندارد.
- چیدمان موبایل بررسی شد و overflow افقی مشاهده نشد.

## معیار پذیرش

- Hero باید در اولین نما پیام محصول را بدون تداخل منتقل کند.
- کاربر باید بداند مسیر اصلی دمو چیست.
- نقش‌های کلیدی اکسپلوریا باید در همان بخش اول قابل فهم باشند.
- صفحه باید در desktop و mobile بدون شکست افقی نمایش داده شود.

## Verification

اجرا شد:

```bash
npm run build
```

نتیجه:

- Vite build موفق بود.
- صفحه در مرورگر داخلی روی desktop بررسی شد.
- صفحه در viewport موبایل `390x844` بررسی شد.
- overflow افقی مشاهده نشد.

نکته:

`npm run types:check` همچنان به خطای قدیمی و غیرمرتبط در `resources/js/pages/dashboard.tsx` متوقف می‌شود و به تغییر landing مربوط نیست.
