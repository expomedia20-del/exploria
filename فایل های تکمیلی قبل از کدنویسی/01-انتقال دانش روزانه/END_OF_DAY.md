# پایان روز کاری EXPLORIA

## چک‌لیست پایان روز

- فهرست فایل‌های تغییرکرده ثبت شود.
- تست‌ها و buildهای اجراشده با نتیجه دقیق ثبت شوند.
- کارهای باز و blockerها نوشته شوند.
- اگر Git آماده بود، commit کوچک با پیام روشن ثبت شود.
- یک ZIP تاریخ‌دار از خروجی روز ساخته شود.
- پوشه انتقال دانش به‌روزرسانی شود.

## قالب گزارش پایان روز

```text
Date:

```

## قواعد بکاپ

- آرشیوهای اصلی قبلی حذف نشوند.
- `node_modules` و `vendor` فقط وقتی لازم است داخل بکاپ قرار بگیرند؛ برای بکاپ کد معمولا حذف شوند.
- `.env` و اطلاعات محرمانه وارد بکاپ قابل انتقال نشوند.
- حداقل یک کپی بیرون از همان پوشه پروژه نگهداری شود.
