# راهنمای ذخیره و بازیابی پروژه اکسپلوریا در Codex

## محل اصلی پروژه

پوشه‌ای که شامل این فایل است، نسخه اصلی و فعال پروژه است. همیشه Codex را روی همین پوشه یا نسخه بازیابی‌شده آن باز کنید.

## لایه‌های نگهداری

1. فایل‌های اصلی در پوشه پروژه ذخیره می‌شوند.
2. Git نسخه‌های تأییدشده را به‌صورت Commit نگهداری می‌کند.
3. نسخه‌های ZIP کامل در پوشه `.codex-backups` قرار می‌گیرند.
4. Thread در Codex فقط سابقه گفت‌وگو است و جایگزین فایل یا Git نیست.

## بازیابی از Git

اگر فایل‌ها به‌اشتباه تغییر کردند، ابتدا در Codex بخواهید وضعیت Git و تاریخچه Commitها بررسی شود. بدون بررسی، از دستورهای حذف یا Reset استفاده نکنید.

## بازیابی از ZIP

1. یک پوشه جدید و خالی بسازید.
2. فایل ZIP منتخب از `.codex-backups` را داخل آن Extract کنید.
3. پوشه استخراج‌شده را به‌عنوان پروژه در Codex باز کنید.
4. از Codex بخواهید سلامت فایل‌ها و وضعیت Git را بررسی کند.
5. تا پایان بررسی، نسخه اصلی و ZIP را حذف نکنید.

## روال پیشنهادی دفعات بعد

- پیش از هر مرحله مهم، از Codex بخواهید یک Commit و ZIP تاریخ‌دار ایجاد کند.
- پس از پایان هر مرحله، دوباره Commit بگیرید.
- حداقل یک نسخه ZIP را خارج از پوشه پروژه، روی فضای ابری یا درایو دیگر نگهداری کنید.
- Thread مهم را نام‌گذاری و Pin کنید؛ Archive کردن Thread پشتیبان فایل‌ها نیست.
